DataBus Fundamentals

Within DataBus there are three main components that define how the data is structured, stored and accessed: Databases, Tables and Modules.

Databases

Databases are top level components that define a set of data in relation to Users, User Groups and Tables. Basically, a database would be the highest level container for a customer to store and access data (although a customer is not limited to just one Database). An example of a Database could be:

Database Name: Western_United_States_Wind_Data
Users: JSmith, BMiller
User Groups: Wind_Research_Group, Statistical_Analysis
Tables: wind103WUV, wind103WTWC500nm, wind103WDirTotal

Tables

Tables describe the data coming from a source of user defined values. A table can be any combination of values that are important to the user defining the table but must fall within two main table types that DataBus offers: STREAM and RELATIONAL_TABLE.

STREAM tables are for time series data. These tables consist of a single TIME column and a single VALUE column.
RELATIONAL_TABLE tables are for relational data. These tables can consist of any number of columns describing the data you want to store.

Modules

Modules are advanced components that allow you to translate data into many different forms when requesting that data from DataBus. Most Modules are specific to a single Table type and cannot be used for data that is stored in a table not of that type. An exception would be the SQL Module which provides SQL-like queries on a table whose structure is known by the one using the module. One powerful feature of the Modules component is that they can be chained together to provide multiple translations on a single request. Some examples of modules are:

Standard Deviation (StdDevMod)Type: STREAM
Splines (SplinesMod)Type: STREAM
SQL Module (SQL)Type: RELATIONAL_TABLE, STREAM


User Roles

There are three distinct end user roles for accessing DataBus: Users, Robots and Groups.

Users are Active Directory system accounts that are allowed access to the DataBus system. A user will be responsible for the creation and maintenance of system components such as Databases, Tables, Robots, User Groups and other Users.

Robots are DataBus system accounts that "act" like Users when running certain DataBus functions. Robots are used to allow access to the system in an automated way without exposing User Credentials for authentication. They should mainly be used for automated services and are considered an Advanced feature of the DataBus system.

User Groups are combinations of Users and Robots that make it easy to give common system privelages to many end user roles at once.


First Time User

As a first time user, you either want access to someone elses data or want to upload your own so we divide this help into two sections. The first section is for those who wish to upload data into the system and use that data. The second is for those who want to look around and find data sets and request access to those datasets.


First Time User to Upload Data

These are generally the first steps you will take as a first time user:

  1. First, you should click My DataBus > Databases
  2. Next, click the button create database
  3. Now that you have a database, you can click Provisioning DataBus to find out how to create tables to upload data to them.

Next, you may want to create a user group: (Optional)

  1. Click on My DataBus > Groups
  2. Click on Create Group
  3. Click save once you have filled the information in
  4. You are automatically an admin member of this group
  5. You can request other users to give your group access to their databases

First Time User to Read Data

  1. The first thing to look through is the list of all databases so click on My DataBus->Summary and scroll down to Databases
  2. Find the database you are interested in and look at the list of admin users or admin groups
  3. If there is an admin user, you can contact that user to request access to that database
  4. If there is only admin groups, you can scroll back up to the Groups section and find the group and look at the Groups admins
  5. Now, you can contact the Group's admins to ask for access to the database

Once you have access, you can move onto the section Downloading Data


Robots

Robots are user accounts that can perform most DataBus functions, just like a normal User and are considered an Advanced Feature of the DataBus System.

The main use for Robot accounts are to interface with the DataBus system in an automated way. Since all DataBus functionality requires authentication, there needed to be a way to perform routine and automated interactions with the system in which User account credentials were not stored.

Functions that are available to Robots:

  • Can create Databases
  • Can create Tables
  • Can insert data into Tables
  • Can update existing Table data
  • Can read existing Table data

How Robots differ from a normal User:

  • Robots only exist as part of a group.
  • Robots access is available to everyone in the group the Robot belongs to.
  • Robots accounts cannot be used to log into the DataBus User Interface.
  • Robots cannot create other Robots or User Groups.

Creating a Robot

To create a Robot, log into your Groups page and click one of the User Groups that you have either created or are associated with. Under the "Members" section, click the "Create robot" link. Enter in the "User Name" of your Robot and make sure that "Member" is the chosen "Membership Type".

Once the Robot is created, you will see it listed in the "Members" section for this particular User Group along with its associated "API Key". When interacting with the DataBus System as a Robot, the Username will be the "Robot Name" and the Password will be the Robot "API Key".

SECURITY NOTE:

Robots only have access to what you give them access to. If a Robot is in a User Group that has full access, the Robot has full access. Robots can be added to other groups via the "Add existing robot to this group" link within a specific User Group page. The owner of the Robot is the User Group selected during Robot creation and is managed by all Admins within that group.

Robots can only be deleted by the Admins of the group the Robot belongs to. Deleting a Robot from a group that the Robot does not belong to, only removes the Robots' membership to that group.


User Groups

A User Group is a DataBus system account that groups Users together in order to make it easy to give common system privilages and data access to many users at once.

Once the group is created, the administrator can then perform the following tasks for the group:

  • Add existing Users to the group.
  • Associate existing Databases with the group.
  • Associate existing Robots with the group.
  • Create and associate new Robots with the group.

When adding a User to a group, that user can have a "Membership Type" of either:

  • Member
  • Group Admin

A Member account can access all data associated with the User Group whereas a Group Admin can create new group Users and new group Robots.

All users that are within a group can access all information about all Databases associated with that group. All users of a group can also access and use all Robots that are associated with the group.

Creating a User Group

To create a User Group, log into the Add User Group Interface and complete the form. The corporate user account that creates the User Group will become the administrator of the group.
